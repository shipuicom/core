A11y Keybindings

Overview & Accessibility

The A11y Keybindings engine provides a unified system for handling keyboard shortcuts. The [shA11yKeybinding] directive dynamically sets standard aria-keyshortcuts attributes on host elements to announce active hotkeys to assistive technologies.

Provider Overrides

Default shortcuts registered by components can be overridden globally at the application level. Provide the SHIP_A11Y_KEYBINDINGS_OVERRIDE injection token with a mapping object: { provide: SHIP_A11Y_KEYBINDINGS_OVERRIDE, useValue: { 'action.name': 'ctrl+s' } }.

OS Adaptive Keys (Command vs Control)

We support the ctrlOrCmd modifier keyword in shortcuts. It maps dynamically at runtime: resolves to Command (⌘) on macOS and Control (Ctrl) on Windows/Linux. Display formats are also customized for each OS (e.g. ⌘⇧K vs Ctrl+Shift+K).

Collisions & Input Handling

When in mode="global", the directive automatically ignores keypresses that originate from input boxes, selects, textareas, or content-editables (unless modifier keys like Ctrl, Cmd, or Alt are pressed), ensuring keyboard shortcuts do not interfere with typing.
