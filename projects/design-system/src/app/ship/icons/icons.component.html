Icons

Sizes

Icons can be set to different sizes using the class class="small" or class="large" or class="inherit"

Colors

Chip colors can be set through the class ex class="primary" takes primary , accent , warn , error and success

Color can also be overridden with css color variables both like this below or in the scss file. {{ `\cloud-warning\` }}

Icon packs

Currently we only support phospher icons, but we are working on adding more.

Search Phosphor Icons arrow-square-out

But we do support follow icon variation types:

  • Regular (default)
  • Thin
  • Light
  • Fill
  • Bold

So the way you access the icon we use ligatures like this:

<sh-icon>cloud-warning</sh-icon>

<sh-icon>cloud-warning-thin</sh-icon>

<sh-icon>cloud-warning-light</sh-icon>

<sh-icon>cloud-warning-fill</sh-icon>

<sh-icon>cloud-warning-bold</sh-icon>

Icon CLI

We have a CLI tool that can help you generate icon fonts so you dont ship all 9000+ icons but only the ones you use.

To generate the icon font you can run the following command:
ship-fg --src='./src' --out='./src/assets' --rootPath='./'


To watch the files and generate the icon font on change you can run the following command: ship-fg --src='./src' --out='./src/assets' --rootPath='./' --watch

  • --src Are the src folder of your code where you use the icons.
  • --out Are the out folder of your project we will generate the icon font here
  • --rootPath Are the root path of your project we will use this to generate the correct path to the icons
  • --watch Are the watch flag we will watch the files and generate the icon font on change

The output will be a woff2 file and a css file that you can use to import the icon font, that just need to be added to your index.html .
